# Adoption rollout

## Week 1: Pilot

Pick 8–12 engineers across experience levels.

Ask each to use `/ppp` on two real tickets.

Capture:

- where it helped
- where it got stuck
- whether it respected repo instructions
- whether PR summaries were good
- whether it reduced copy/paste chat usage
- whether engineers understood the final change

## Week 2: Improve

Patch the skill and docs based on pilot feedback.

Add repo-specific PR/commit conventions to key repos.

Add examples of good PPP-created PRs.

## Week 3: Team rollout

Run a 45-minute live demo:

1. bad workflow: paste snippet from chat
2. good workflow: `/ppp` in IDE
3. use recommended answers
4. test fails once
5. review/hardening step
6. PR handoff
7. explain changes before committing

## Week 4+: Measure

Track:

- installs / active usage
- PR review comments about AI mistakes
- rework rate
- PR cycle time
- dev sentiment
- examples of good PPP-created PRs
- repeated failure modes

## Success indicators

PPP is working when:

- engineers stop using AI as a detached snippet generator
- PRs include clearer testing notes
- review comments shift from basic correctness to higher-level design/product concerns
- engineers can explain AI-assisted changes
- agents stop instead of guessing on high-risk work
